# Lab 5: Intro to Spec-Driven Development

## 🎯 Challenge Overview

**Time Limit**: 30 minutes  
**Difficulty**: Intermediate to Advanced  
**Type**: Open-ended Challenge

Build a weather application using **spec-driven development**. Define your specifications first, then build to meet them.

## 🎪 The Challenge

Create a weather application with both backend and frontend components.

## 📋 Spec-Driven Approach

### Phase 1: Define Specifications (5-10 min)

Create two specification documents:

**`FUNCTIONAL_SPEC.md`** - What your app does:
- What problem does it solve?
- What features will it have?
- What user flows are supported?
- What data is displayed and how?

**`TECHNICAL_SPEC.md`** - How your app works:
- Which weather API will you use?
- What tech stack (backend/frontend)?
- What architecture design?
- What authentication/API key method?
- What endpoints will your backend expose?
- How will you handle errors and security?

**💡 Use Bob to help you think through and document your specifications!**

### Phase 2: Build to Spec (15-20 min)

- Implement according to your specifications
- Test against your requirements
- Document any spec changes and why

### Phase 3: Validate (5 min)

- Verify implementation matches specifications
- Document deviations

## 💡 Tips

- Start with specifications before coding
- Keep specs realistic for 30 minutes
- Use Bob's different modes strategically
- Simple solutions are often best

## 🤔 Reflection

After completing:
1. Did specs help or hinder your development?
2. What would you specify differently next time?
3. How did Bob help with spec creation vs. implementation?
